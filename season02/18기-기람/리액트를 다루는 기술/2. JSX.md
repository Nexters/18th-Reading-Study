### 2. JSX

##### 2.1 코드 이해하기

- ```javascript
  import React from 'react';
  ```

  create-react-app으로 리액트 프로젝트를 만들 때 node_modules라는 디렉터리가 생성되고 이 안에 react 모듈이 설치됨. 때문에 import 구문을 통해 리액트를 불러와서 사용할 수 있음

- 모듈을 불러와서 사용하는 것은 원래 브라우저에 없음. Node.js 에서 지원하는 기능임

- 이런 기능을 브라우저에서도 사용하기 위해 번들러를 사용

- 번들러의 종류

  - 웹팩 (리액트에서 주로 씀)
  - Parcel
  - browserify

- 번들러를 쓰면 import(또는 require)로 모듈을 불러왔을 때 불러온 모듈을 모두 합쳐서 하나의 파일 생성



##### 2.2 JSX

- JS의 확장 문법
- 브라우저에서 실행되기 전에 코드가 번들링되는 과정에서 바벨을 사용하여 일반 JS 형태로 변환됨
- 장점
  - 보기 쉽고 익숙함(html)
  - 활용도가 높음(컴포넌트에서도 JSX를 쓸 수 있음)
- 문법
  - 여러 개의 요소를 하나의 요소로 감싸 주어야 함
    - Why? Virtual DOM에서 컴포넌트 변화를 감지해 낼 때, 효율적으로 비교할 수 있도록 컴포넌트 내부는 하나의 DOM트리 구조로 이루어져야하기 때문
  - JS 표현식을 쓰려면 {}로 감싸면 됨



##### Note

- ReactDOM.render란? 컴포넌트를 페이지에 렌더링하는 역할
- React.StrictMode란? 리액트 프로젝트에서 리액트의 레거시 기능을 사용하지 못하게 하는 기능
- var & const & let
  - var의 scope : 함수 단위
  - const / let의 scope : 블록 단위



##### 2.4 문법

- 컴포넌트에서 여러 개의 요소를 꼭 하나로 감싸 줘야 하는 이유

  - Virtual DOM에서 컴포넌트 변화를 감지해 낼 때 효율적으로 비교할 수 있도록 컴포넌트 내부는 하나의 DOM 트리 구조로 이루어져야하는 규칙때문
